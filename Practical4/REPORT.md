# REPORT.md
1. Що таке незмінність рядків у C#?
Незмінність рядків означає, що об’єкт типу string після створення не змінюється. Якщо до рядка додати новий текст, C# не змінює старий рядок, а створює новий об’єкт у пам’яті.
Через це багаторазова конкатенація в циклі працює повільно, бо кожного разу створюється новий рядок. Цю проблему вирішує StringBuilder, який дозволяє ефективно накопичувати текст без постійного створення нових об’єктів.
У моїй програмі це видно в методі ComparePerformance, де порівнюється звичайний string і StringBuilder.

2. У яких місцях програми застосовано StringBuilder?
StringBuilder використано у тих місцях, де формується великий або багато рядковий текст.
У моїй програмі він застосований:
у AdvancedLogger для накопичення логів;
у Student.GetFormattedInfo() для формування картки студента;
у StudentGroup.SearchByNameFragment() для результатів пошуку;
у StudentGroup.ExportToCsv() для створення CSV;
у StudentGroup.GenerateBigReport() для великого звіту;
у TextProcessor.BuildGroupReport() для повного звіту групи;
у MoodAnalyzer.AnalyzeGroupMood() для текстового результату аналізу настрою.
Це краще за звичайну конкатенацію, бо ці методи багато разів додають нові рядки до одного тексту.

3. Результати порівняння продуктивності
У програмі реалізовано метод:
ComparePerformance(int iterations)
Він порівнює додавання тексту через string та через StringBuilder. Для перевірки використовується клас Stopwatch.
Приклад результатів:
1000 ітерацій:
string — 2 ms
StringBuilder — 0 ms
10000 ітерацій:
string — 150 ms
StringBuilder — 1 ms
100000 ітерацій:
string — значно повільніше
StringBuilder — працює швидше
Точні значення можуть відрізнятися залежно від комп’ютера, але загальний результат показує, що StringBuilder ефективніший для великої кількості операцій додавання тексту.

4. Які методи класу String використовувалися найбільше?
У роботі найчастіше використовувалися такі методи рядків:
Trim() — для видалення зайвих пробілів на початку і в кінці рядка. Наприклад, у валідації ПІБ студента та імпорті студентів з тексту.
Split() — для розбиття рядка на частини. Він використовується у FullName, ImportStudentsFromText, CountWords, Normalize, SplitIntoSentences.
Contains() — для пошуку тексту, наприклад у ContainsKeyword, FindByName, SearchByNameFragment.
Replace() — для видалення пробілів у CountCharacters та IsPalindrome.
ToLower() і ToUpper() — для приведення тексту до одного регістру, наприклад у логах і аналізі настрою.
Join() — у методі Normalize, щоб зібрати очищені слова назад в один рядок.

5. Пояснення IsPalindrome та Normalize
Метод Normalize очищає текст від зайвих пробілів. Спочатку він перевіряє, чи рядок не порожній. Потім розбиває текст методом Split, прибирає порожні елементи і з’єднує слова назад через один пробіл за допомогою string.Join.
Метод IsPalindrome перевіряє, чи однаково читається текст зліва направо і справа наліво. Спочатку текст обробляється: за потреби прибираються пробіли через Replace, а також текст переводиться в нижній регістр через ToLower. Після цього рядок розвертається методом Reverse, і результат порівнюється з початковим обробленим текстом.

6. Як інтегровано функціонал рядків у Student та StudentGroup?
У клас Student було додано метод GetFormattedInfo, який формує інформацію про студента через StringBuilder. Також було додано метод ContainsKeyword, який шукає ключове слово у ПІБ, email та нотатках студента. Крім цього, була посилена валідація fullName: ПІБ має містити щонайменше три слова.
У клас StudentGroup додано пошук студентів за фрагментом ПІБ, експорт групи в CSV та імпорт студентів із текстового блоку. Це дозволило інтегрувати роботу з рядками безпосередньо в систему управління групою.

7. Які труднощі виникали?
Під час виконання роботи виникали труднощі з правильною обробкою тексту, тому що користувач може вводити зайві пробіли, порожні рядки або некоректний формат даних.
Для вирішення цих проблем було використано:
Trim() для очищення тексту;
Split() з RemoveEmptyEntries;
перевірки string.IsNullOrWhiteSpace;
блоки try/catch під час імпорту студентів;
валідацію ПІБ, email і номера залікової книжки.
Також складність була у формуванні великих звітів, тому для цього використано StringBuilder.

8. Що нового було вивчено?
Під час виконання практичної роботи я краще зрозуміла різницю між string і StringBuilder. Також навчилася ефективніше працювати з текстовими даними: розбивати рядки, очищати їх, шукати фрагменти, замінювати слова, перевіряти паліндроми та формувати звіти.
Окремо було закріплено використання StringBuilder для логів, CSV-файлів, звітів і великих текстових результатів. Крім цього, було інтегровано текстову обробку в уже існуючу систему управління студентською групою.
