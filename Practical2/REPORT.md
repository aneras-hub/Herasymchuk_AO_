1. Поясніть різницю між одновимірними та двовимірними масивами. Наведіть приклади використання в роботі.
Одновимірний масив — це структура даних, яка зберігає елементи в одному рядку. До кожного елемента можна звернутися за одним індексом. У моїй роботі одновимірний масив використовується в класі Student для зберігання оцінок за лабораторні роботи:
public byte[] LabGrades { get; set; } = new byte[10];
Цей масив містить 10 елементів, тобто може зберігати оцінки за 10 лабораторних робіт. Для роботи з ним створені методи AddLabGrade(), GetAverageLabGrade() та SortLabGrades().
Двовимірний масив — це масив, який має два індекси: рядок і стовпець. Його зручно уявити як таблицю або матрицю. У моїй роботі двовимірний масив використовується в класі PortMatrix:
C#
private Port[,] ports;
Ця матриця має розмір 16×16 і зберігає об’єкти класу Port. Кожен порт має свої координати: рядок і стовпець.

2. Чому StringBuilder ефективніший за звичайну конкатенацію? Продемонструйте на прикладі з PortLogger.
StringBuilder ефективніший за звичайну конкатенацію рядків, тому що не створює щоразу новий об’єкт рядка при додаванні тексту. Звичайні рядки в C# є незмінними, тому при кожному використанні оператора + створюється новий рядок у пам’яті. Якщо текст формирується багато разів, це може бути неефективно.
У моїй роботі StringBuilder використовується в класі PortLogger для накопичення логів:
C#
private readonly StringBuilder logBuilder = new();public void LogOperation(string operation, int portNumber, string details)
{
    logBuilder.AppendLine(
        $"[{DateTime.Now: HH:mm:ss}] " +
        $"{operation} | " +
        $"Port: {portNumber} | " +
        $"{details}"
    );
}
Кожна операція з портом додається до одного спільного об’єкта StringBuilder. Це зручно для логування, тому що записів може бути багато. Потім увесь лог можна отримати методом:
C#
public string GetFullLog(){
    return logBuilder.ToString();}
Також StringBuilder використовується у методах ShowDetailedInfo(), ScanMatrix(), GetPortInfo() та GenerateBigReport().

3. Як ви реалізували симуляцію портів вводу/виводу? Які масиви використовували?
Симуляція портів вводу/виводу реалізована через класи Port, PortMatrix та PortLogger.
Клас Port представляє один порт. Він має номер, назву пристрою, стан відкритий/закритий і буфер даних:
C#
public byte[] DataBuffer { get; }
DataBuffer — це одновимірний масив байтів на 64 елементи. Він використовується для імітації запису даних у порт.
У класі Port реалізовані методи:
Open()
Close()
WriteData(byte[] data)
ReadData()
GetPortInfo()
Метод WriteData() записує дані в буфер тільки тоді, коли порт відкритий. Якщо порт закритий, генерується виняток PortException.
Для зберігання всіх портів використовується клас PortMatrix, у якому створено двовимірний масив:
C#
private Port[,] ports;
Ця матриця має розмір 16×16. Кожна комірка містить окремий порт. Через методи OpenPort(), ClosePort(), WriteToPort() та ReadFromPort() можна керувати портами за координатами.

4. Як інтегрували масиви в already існуючу систему управління студентами?
Масиви були інтегровані в систему з Практичної №1 через розширення класу Student та класу StudentGroup.
У клас Student було додано одновимірний масив:
C#
public byte[] LabGrades { get; set; } = new byte[10];
Він зберігає оцінки студента за лабораторні роботи. Також були додані методи:
AddLabGrade(int labNumber, byte grade)
GetAverageLabGrade()
SortLabGrades()
Для зв’язку студента з портом у класі Student додано координати:
C#
public int PortRow { get; set; } = -1;public int PortCol { get; set; } = -1;
У класі StudentGroup було додано об’єкти:
C#
private PortMatrix portMatrix = new();private PortLogger logger = new();
Метод AssignStudentToPort() прив’язує студента до конкретного порту. Метод SimulateLabWork() імітує виконання лабораторної роботи: студент отримує оцінку в масив LabGrades, а дані записуються в порт. Після цього операція фіксується в логах через PortLogger.

5. Опишіть логіку роботи з матрицею 16×16
Матриця портів реалізована в класі PortMatrix. Вона зберігає 256 портів, тому що розмір матриці становить 16×16.
Під час створення об’єкта PortMatrix викликається метод InitializeMatrix(). Він проходить по всіх рядках і стовпцях матриці та створює об’єкти Port:
C#
for (int row = 0; row < 16; row++)
{
    for (int col = 0; col < 16; col++)
    {
        ports[row, col] = new Port(portNumber, $"Device-{portNumber}");
        portNumber++;
    }
}
Кожен порт отримує унікальний номер і назву пристрою. Доступ до порту здійснюється за координатами row і col.
Перед роботою з портом виконується перевірка індексів у методі ValidateIndexes(). Якщо користувач вводить неправильний рядок або стовпець, програма генерує IndexOutOfRangeException.
Матриця підтримує такі операції:
відкриття порту;
закриття порту;
запис даних у порт;
читання даних з порту;
виведення стану всієї матриці;
пошук відкритих портів за назвою пристрою.
Метод ScanMatrix() формує повний стан матриці у вигляді тексту через StringBuilder.

6. Які труднощі виникали при роботі з масивами та як ви їх вирішили?
Першою складністю була робота з індексами масивів. У масивах індексація починається з нуля, тому для масиву лабораторних робіт на 10 елементів допустимі індекси від 0 до 9. Щоб уникнути помилок, у методі AddLabGrade() додано перевірку:
C#
if (labNumber < 0 || labNumber >= LabGrades.Length)
    throw new IndexOutOfRangeException(...);
Другою складністю була робота з матрицею портів. Для двовимірного масиву потрібно правильно перевіряти одразу два індекси: рядок і стовпець. Для цього створено окремий метод ValidateIndexes().
Третьою складністю було те, що порт не повинен приймати дані, якщо він закритий. Це вирішено у методі WriteData(), де перед записом перевіряється IsOpen.
Четвертою складністю було формування великої кількості текстової інформації. Для цього замість конкатенації рядків було використано StringBuilder.
Ще одна складність виникла при інтеграції портів зі студентами. Потрібно було зробити так, щоб студент міг бути прив’язаний до певного порту. Це вирішено через властивості PortRow і PortCol у класі Student.

7. Порівняйте масиви з колекціями List<T>. У яких випадках ви свідомо обрали масиви?
Масиви і колекції List<T> обидва використовуються для зберігання набору даних, але мають різне призначення.
Масив має фіксований розмір. Якщо створити масив на 10 елементів, він завжди матиме саме 10 елементів. Це зручно, коли кількість даних відома заздалегідь. У моїй роботі масив використано для лабораторних оцінок, тому що кількість лабораторних обмежена:
C#
public byte[] LabGrades { get; set; } = new byte[10];
Також масив використано для буфера порту:
C#
public byte[] DataBuffer { get; }
Тут теж розмір фіксований — 64 байти.
Двовимірний масив використано для матриці портів:
C#
private Port[,] ports;
Це логічно, тому що матриця портів має чіткий розмір 16×16.
List<T> краще підходить тоді, коли кількість елементів може змінюватися. У моїй роботі список використано для студентів:
C#
private List<Student> _students = new();
Кількість студентів у групі може збільшуватися або зменшуватися, тому для них зручніше використовувати List<Student>.
Отже, масиви я обрала там, де розмір відомий наперед, а List<T> — там, де кількість елементів змінюється.

8. Що нового ви дізналися під час виконання роботи?
Під час виконання цієї практичної роботи я краще розібралася з одновимірними та двовимірними масивами в C#. Я навчилася використовувати одновимірний масив для зберігання оцінок за лабораторні роботи та двовимірний масив для створення матриці портів 16×16.
Також я навчилася працювати з байтовими масивами. У класі Port байтовий масив DataBuffer використовується для зберігання даних, які записуються в порт.
Окремо я зрозуміла, чому StringBuilder краще підходить для формування великих текстів, логів і звітів. Він використовується в PortLogger, у виведенні інформації про студента, у скануванні матриці портів і при створенні великого звіту.
Також я навчилася інтегрувати новий функціонал у вже існуючу систему. До системи студентів було додано лабораторні оцінки, порти, матрицю портів, логування операцій і симуляцію лабораторної роботи.
Крім цього, я попрактикувалася в обробці винятків, наприклад при неправильних індексах масиву, закритому порті або некоректних даних. Після виконання роботи я краще зрозуміла, як розділяти програму на окремі класи та як поєднувати їх між собою.
