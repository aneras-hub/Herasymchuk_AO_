# REPORT.md
# 1. Що таке поліморфізм? Які види поліморфізму існують у C#?
Поліморфізм — це один із основних принципів об’єктно-орієнтованого програмування, який дозволяє працювати з різними об’єктами через спільний базовий тип. Завдяки поліморфізму одна й та сама змінна або колекція може містити об’єкти різних класів, а виклик методів буде виконуватись відповідно до реального типу об’єкта.
У C# існують такі види поліморфізму:
* перевантаження методів (compile-time polymorphism);
* перевантаження операторів;
* поліморфізм через наслідування ( virtual  /  override );
* поліморфізм через інтерфейси.
У практичній роботі використовувався поліморфізм через наслідування та інтерфейси для роботи з геометричними фігурами та платежами.

# 2. Поясніть роботу ключових слів virtual та override. Наведіть приклади з коду.
Ключове слово  virtual  використовується в базовому класі для позначення методу, який можна перевизначити в похідних класах.
Ключове слово  override  використовується в похідному класі для зміни реалізації віртуального методу.
Приклад із програми:
public virtual double CalculateArea()
{
    return 0;
}
У похідному класі:
public override double CalculateArea()
{
    return Math.PI * Radius * Radius;
}
У класах  Circle ,  Rectangle ,  Triangle  та  Square  були перевизначені методи:
*  CalculateArea() 
*  CalculatePerimeter() 
*  GetDescription() 

# 3. У чому різниця між абстрактним класом і інтерфейсом? Де ви використали кожен?
Абстрактний клас використовується як базовий шаблон для інших класів. Він може містити як звичайні методи, так і абстрактні. Об’єкти абстрактного класу створювати не можна.
Інтерфейс описує лише набір методів, які клас повинен реалізувати.
У роботі:
* абстрактний клас  Shape  використовувався як базовий клас для всіх геометричних фігур;
* інтерфейси  IResizable ,  IDrawable ,  IPrintable  використовувались для реалізації додаткової поведінки фігур.
Наприклад:
*  IResizable  — зміна розміру;
*  IDrawable  — симуляція малювання;
*  IPrintable  — виведення інформації.

# 4. Що таке динамічне (пізнє) зв’язування? Як воно реалізується в вашій програмі?
Динамічне зв’язування — це механізм, при якому виклик методу визначається під час виконання програми, а не під час компіляції.
У програмі це реалізується через використання базового типу  Shape :
foreach (Shape shape in student.Shapes)
{
    Console.WriteLine(shape.GetDescription());
}
Незважаючи на те, що змінна має тип  Shape , викликається метод конкретного класу:
*  Circle 
*  Rectangle 
*  Triangle 
*  Square 
Саме це і є динамічним зв’язуванням.

# 5. Наведіть приклад поліморфної колекції та виклику методів через базовий тип.
У програмі була створена поліморфна колекція:
List<Shape> Shapes
У неї додавались різні типи фігур:
student.Shapes.Add(new Circle(...));
student.Shapes.Add(new Rectangle(...));
student.Shapes.Add(new Triangle(...));
student.Shapes.Add(new Square(...));
Виклик методів здійснювався через базовий тип:
foreach (Shape shape in student.Shapes)
{
    Console.WriteLine(shape.GetDescription());
}
 
# 6. Як поліморфізм покращив архітектуру вашої системи?
Поліморфізм дозволив зробити систему більш гнучкою та масштабованою. Усі фігури обробляються через один базовий тип  Shape , тому для додавання нової фігури не потрібно змінювати вже існуючий код.
Також завдяки інтерфейсам стало можливим реалізувати спільну поведінку для різних класів:
* масштабування;
* малювання;
* друк інформації.
Це спростило структуру програми та зменшило дублювання коду.

# 7. Які труднощі виникали під час реалізації? Як їх вирішували?
Під час виконання роботи виникали такі труднощі:
* помилки через дублювання методів;
* проблеми з інтерфейсами ( GetPrintInfo );
* помилки серіалізації абстрактного класу  Shape ;
* поява значення  NaN  при некоректних сторонах трикутника.
Для вирішення проблем:
* були видалені дублікати методів;
* реалізовані всі методи інтерфейсів;
* для  Shapes  використано  [JsonIgnore] ;
* у класі  Triangle  додано перевірку коректності сторін трикутника.
