# REPORT.md
1. Що таке перевантаження операторів? Які оператори можна, а які не можна перевантажувати в C#?
Перевантаження операторів — це можливість змінити поведінку стандартних операторів для власних класів. Завдяки цьому об’єкти користувацьких типів можна використовувати так само зручно, як і стандартні типи даних.
У C# можна перевантажувати:
арифметичні оператори (+, -, *, /, %);
оператори порівняння (>, <, >=, <=, ==, !=);
унарні оператори (++, --, !);
оператори приведення типів (implicit, explicit);
оператори true та false.
Не можна перевантажувати:
оператор присвоєння =;
оператор доступу . ;
?:;
new;
is;
typeof.
У моїй програмі перевантаження операторів використовується для класів Student, StudentGroup, Vector, GradePoint та Fraction.

2. Приклади перевантажених операторів з коду
У програмі було реалізовано багато перевантажених операторів:
Student > Student
Student < Student
Student == Student
Student + Student
StudentGroup + StudentGroup
Vector + Vector
Vector * Vector
GradePoint ++
GradePoint true/false
Fraction + Fraction
Оператори використовуються для порівняння студентів, об’єднання груп, роботи з векторами, оцінками та дробами.

3. Різниця між явним та неявним приведенням типів
Неявне приведення (implicit) виконується автоматично без додаткових дій програміста.
Явне приведення (explicit) потребує ручного приведення типу через дужки.
У моїй програмі:
для GradePoint реалізовано неявне приведення GradePoint ↔ double;
для Vector реалізовано явне приведення до double, яке повертає довжину вектора.
Приклад:
double value = g1;
double len = (double)v1;
Неявне приведення використовується там, де операція є безпечною, а явне — коли можливі втрати інформації або додаткові обчислення.

4. Для чого потрібні оператори true та false?
Оператори true та false дозволяють використовувати об’єкти класу в умовах if.
У моїй програмі вони реалізовані в класі GradePoint.
public static bool operator true(GradePoint g)
{
    return g.Value >= 8;
}
public static bool operator false(GradePoint g)
{
    return g.Value < 8;
}
Тому оцінку можна використовувати прямо в if:
if (g2)
{
    Console.WriteLine("Оцінка висока");
}
У моїй реалізації оцінка вважається true, якщо вона більша або дорівнює 8.

5. Як реалізовано порівняння студентів?
Для порівняння студентів використано оператори:
>, <, >=, <=, ==, !=.
Порівняння виконується за:
середнім балом;
прогресом навчання (CourseProgress).
Для цього створено метод:
CompareStudents(Student a, Student b)
Він повертає:
1, якщо перший студент кращий;
-1, якщо другий;
0, якщо вони рівні.
Після цього оператори використовують цей метод для порівняння.

6. Які труднощі виникали?
Основна складність була в правильній реалізації пар операторів.
Наприклад:
якщо перевантажується ==, потрібно також перевантажити !=;
якщо перевантажується >, потрібно перевантажити <.
Також виникли труднощі з:
перевіркою null;
перевизначенням Equals() та GetHashCode();
приведенням типів;
коректною роботою true/false.
Для вирішення цих проблем використовувалися:
ReferenceEquals;
перевірки is null;
метод HashCode.Combine;
окремі допоміжні методи для порівняння.

7. Як перевантаження операторів покращує читабельність коду?
Перевантаження операторів робить код більш компактним і зрозумілим.
Наприклад:
if (student1 > student2)
виглядає набагато краще, ніж:
if (student1.CompareTo(student2) > 0)
Так само:
group1 + group2
зручніше для читання, ніж окремий метод об’єднання.
Завдяки цьому код більше нагадує роботу зі стандартними типами даних.
