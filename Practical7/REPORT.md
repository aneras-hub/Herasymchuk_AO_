# REPORT.md
1. У чому полягають ключові відмінності між class і struct?
Основна відмінність між  class  і  struct  у C# полягає в тому, що класи є reference types, а структури — value types.
Об’єкти класів зберігаються в heap-пам’яті та передаються за посиланням. При копіюванні класу копіюється лише посилання на об’єкт.
Структури зберігаються переважно у stack-пам’яті та передаються за значенням. При копіюванні структури створюється повна копія всіх її полів.
Також:
* структури не підтримують наслідування;
* структури не можуть мати деструктор;
* структури не можуть наслідувати інші структури або класи;
* структури мають менші витрати пам’яті.

2. Коли доцільно використовувати структури, а коли класи?
Структури доцільно використовувати для невеликих об’єктів, які:
* містять мало даних;
* часто створюються;
* не потребують наслідування;
* повинні працювати швидко та економно по пам’яті.
У роботі структурами були:
*  Point 
*  GradeRecord 
*  StudentRecord 
*  ComplexNumber 
*  Money 
Класи доцільно використовувати для складних об’єктів із великою кількістю логіки та взаємозв’язків.
У системі класами залишилися:
*  Student 
*  StudentGroup 
*  Port 
*  Vehicle 
*  Shape 

3. Що таке readonly struct і які переваги він дає?
 readonly struct  — це структура, поля якої не можуть змінюватися після створення об’єкта.
Приклад: public readonly struct Point
Переваги:
* захист від випадкової зміни даних;
* покращення продуктивності;
* зменшення кількості копіювань;
* безпечніша робота зі структурами.
У практичній роботі  readonly struct  використовувався для:
*  Point 
*  GradeRecord 
*  StudentRecord 
*  ComplexNumber 
*  Money 

4. Наведіть приклади реалізації Equals(), GetHashCode() та IEquatable<T> у ваших структурах.
У структурах було реалізовано інтерфейс  IEquatable<T>  для коректного порівняння об’єктів.
Приклад із  Point :    
public readonly struct Point : IEquatable<Point>
Метод  Equals() :
public bool Equals(Point other)
{
    return Row == other.Row &&
           Column == other.Column;
}
Метод  GetHashCode() :   
public override int GetHashCode()
{
    return HashCode.Combine(Row, Column);
}
Оператори:   
public static bool operator ==(Point left, Point right)
{
    return left.Equals(right);
}
public static bool operator !=(Point left, Point right)
{
    return !(left == right);
}
   
5. Які результати ви отримали при порівнянні продуктивності struct vs class? Проаналізуйте їх.
Для порівняння було створено клас  PerformanceTest , який генерував:
* масив структур  StudentRecord[] ;
* масив класів  Student[] .
Порівнювалися:
* швидкість заповнення;
* сортування;
* пошук;
* споживання пам’яті.
Результати показали, що структури:
* споживають менше пам’яті;
* швидше працюють при великій кількості дрібних об’єктів;
* створюють менше навантаження на Garbage Collector.
Класи виявилися гнучкішими, але менш ефективними при масових операціях.

6. Як інтеграція структур вплинула на вашу систему?
Інтеграція структур дозволила оптимізувати систему управління студентами.
Було додано:
*  StudentRecord  для швидкого копіювання даних студентів;
*  GradeRecord  для історії оцінок;
*  Point  для координат лабораторних місць.
У  StudentGroup  були реалізовані:
*  OptimizeStorage() 
*  GetAllRecords() 
Завдяки цьому система стала:
* ефективнішою по пам’яті;
* швидшою при обробці великих масивів даних;
* більш структурованою.

7. Які труднощі виникали під час роботи зі структурами?
Під час виконання роботи виникали такі труднощі:
* правильна реалізація  Equals()  та  GetHashCode() ;
* розуміння різниці між копіюванням класів і структур;
* помилки при порівнянні структур;
* проблеми з boxing;
* інтеграція структур у вже існуючу систему.
Для вирішення проблем:
* використовувався  IEquatable<T> ;
* були перевизначені оператори  ==  та  != ;
* використовувався  readonly struct ;
* проводилося тестування через консольне меню.

8. Що нового ви дізналися під час виконання цієї практичної роботи?
Під час виконання практичної роботи було отримано практичні навички:
* роботи зі структурами;
* використання  readonly struct ;
* реалізації  IEquatable<T> ;
* перевизначення  Equals()  і  GetHashCode() ;
* роботи з value types;
* порівняння продуктивності  struct  та  class ;
* оптимізації пам’яті;
* інтеграції структур у великі проєкти.
Також було отримано досвід використання структур для оптимізації вже існуючої системи управління студентами.
