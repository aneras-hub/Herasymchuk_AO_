1. Що таке делегат у C#? Чим він схожий і чим відрізняється від вказівника на функцію?
Делегат у C# — це спеціальний тип даних, який зберігає посилання на метод. Делегати дозволяють передавати методи як параметри, викликати їх динамічно та реалізовувати механізм подій.
Схожість із вказівником на функцію:
•	обидва зберігають посилання на метод;
•	дозволяють викликати функції непрямим способом.
Відмінності:
•	делегати є типобезпечними;
•	можуть зберігати кілька методів одночасно;
•	підтримують події (event);
•	працюють у межах об’єктно-орієнтованої моделі C#.
У роботі були створені власні делегати:
public delegate void StudentOperation(Student student);
public delegate void GroupOperation(StudentGroup group);

2. Поясніть різницю між делегатом та подією (event)
Делегат — це об’єкт, який може зберігати посилання на методи та викликати їх.
Подія (event) — це спеціальний механізм над делегатами, який обмежує доступ до виклику делегата лише всередині класу.
Основні відмінності:
•	делегат можна викликати з будь-якого місця;
•	подію можна лише підписувати або відписувати;
•	події використовуються для реалізації реакції на дії користувача або зміни стану системи.
У роботі події використовувались для:
•	додавання студента;
•	видалення студента;
•	зміни оцінок;
•	генерації звітів.
Приклад:
public event EventHandler<StudentEventArgs> StudentAdded;
public event EventHandler<StudentEventArgs> StudentRemoved;
public event EventHandler<StudentEventArgs> GradeChanged;

3. Наведіть приклади використання Action<>, Func<> та Predicate<> у вашому коді.
Predicate
Використовувався для фільтрації студентів:
public List<Student> FilterStudents(Predicate<Student> predicate)
{
    return _students.FindAll(predicate);
}
Приклад виклику:
List<Student> excellentStudents =
    group.FilterStudents(
        s => s.averageGrade >= 90
    );
Func<Student, double>
Використовувався для обчислення показників студента:
public double CalculateStudentMetric(
    Student student,
    Func<Student, double> calculator)
{
    return calculator(student);
}
Приклад:
double average =
    group.CalculateStudentMetric(
        student,
        s => s.averageGrade
    );
Action
Використовувався для логування:
public void ExecuteLogAction(
    string message,
    Action<string> logger)
{
    logger(message);
}
Приклад:
advancedLogger.ExecuteLogAction(
    "Тестове повідомлення",
    message =>
    {
        Console.WriteLine($"[LOG] {message}");
    }
);

4. Як працює багатовіщальний делегат? Де ви його застосовували?
Багатовіщальний делегат дозволяє одночасно викликати кілька методів через один делегат.
У роботі це було реалізовано через події:
notificationSystem.StudentAdded += (sender, e) =>
{
    Console.WriteLine($"[LOG] {e.Message}");
};
notificationSystem.StudentAdded += (sender, e) =>
{
    Console.WriteLine($"[EMAIL] Повідомлення про студента {e.Student.FullName}");
};
Після виклику події виконувались обидва обробники.

5. Поясніть механізм лямбда-виразів. Наведіть 3–4 приклади з роботи.
Лямбда-вирази — це скорочений спосіб запису анонімних функцій.
Вони використовуються разом із делегатами, LINQ та подіями.
Приклади:
Фільтрація студентів
s => s.averageGrade >= 90
Обчислення середнього балу
s => s.averageGrade
Сортування студентів
s => s.FullName
Callback-функція
student =>
{
    Console.WriteLine(student.FullName);
}

6. Як ви використовували події в системі студентів?
У системі студентів події використовувались для автоматичного реагування на зміни в групі.
Було реалізовано:
•	StudentAdded
•	StudentRemoved
•	GradeChanged
•	AverageGradeChanged
Події викликались при:
•	додаванні студента;
•	видаленні студента;
•	зміні оцінок;
•	зміні середнього балу.
Приклад:
StudentAdded?.Invoke(
    this,
    new StudentEventArgs(
        student,
        $"Студента {student.FullName} додано."
    )
);

7. Які труднощі виникали під час роботи з делегатами та як їх вирішували?
Під час виконання роботи виникали такі труднощі:
•	дублювання методів;
•	неправильне розміщення подій у класах;
•	помилки з областями видимості;
•	неправильне закриття фігурних дужок;
•	помилки типів під час роботи з Func та Action.
Для вирішення проблем:
•	було розділено логіку між Student, StudentGroup та NotificationSystem;
•	події перенесено у відповідні класи;
•	перевірено правильність сигнатур делегатів;
•	використано лямбда-вирази для спрощення коду.

8. Що нового ви дізналися під час виконання цієї практичної роботи?
Під час виконання практичної роботи було вивчено:
•	створення власних делегатів;
•	використання Action<>, Func<>, Predicate<>;
•	механізм подій (event);
•	багатовіщальні делегати;
•	лямбда-вирази;
•	callback-механізми;
